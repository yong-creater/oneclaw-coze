# TestCraft - AI 测试用例生成平台

## 产品需求文档 (PRD)

---

## 1. 产品概述

### 1.1 产品定位
TestCraft 是一款 AI 驱动的测试用例自动生成与管理平台，专注于帮助测试团队快速将需求文档转化为标准化、可执行的测试用例。

### 1.2 核心价值
- **效率提升**：将需求分析到用例生成的周期从数天缩短到分钟级
- **标准化输出**：基于 BDD 格式的测试用例，符合行业规范
- **全程可追溯**：用例与需求点一一对应，支持需求追溯
- **多格式导出**：支持 CSV、Excel、XMind 等主流格式

### 1.3 目标用户
- 软件测试工程师
- 产品经理（编写验收标准）
- QA 负责人（管理测试用例库）
- 开发团队（理解需求和验收标准）

## 3. 功能详细说明

### 3.1 需求输入模块

#### 3.1.1 手动输入
- **需求标题**：必填，最长 100 字符
- **需求描述**：支持多行文本输入，描述完整的功能需求
- **模块选择**：星链、C端APP、工作手机（单选）
- **AI 模型选择**：豆包 Seed 2.0 Pro、DeepSeek 等

#### 3.1.2 附件上传
- **支持格式**：PDF、Word (doc/docx)、HTML、TXT
- **上传流程**：
  1. 用户拖拽或点击上传文件
  2. 文件上传到 S3 对象存储
  3. 自动解析文档内容
  4. 展示在"已解析内容"预览区
- **解析能力**：
  - PDF：提取文本内容
  - Word：提取段落和表格
  - HTML：提取正文文本
  - TXT：直接读取

#### 3.1.3 链接抓取
- **输入格式**：支持 HTTP/HTTPS URL
- **智能处理**：
  - 自动识别网页编码
  - 提取正文内容
  - 过滤广告和导航
- **限制提示**：
  - 飞书、Notion 等需登录的平台显示友好提示
  - 超时限制 15 秒
  - 最大内容长度 50000 字符

#### 3.1.4 知识库集成
- **树形目录结构**：左侧导航 + 右侧用例列表
- **文件夹选择**：支持文件夹级联选择，自动关联所有子用例
- **搜索功能**：支持关键词搜索相似用例
- **相似度展示**：显示用例与当前需求的匹配度

### 3.2 需求拆分模块

#### 3.2.1 AI 自动拆分
- **调用接口**：POST /api/analyze-requirements
- **输出格式**：JSON 结构化数据
- **拆分规则**：
  - 按功能模块拆分
  - 按用户角色拆分
  - 按业务流程拆分
  - 按操作场景拆分

#### 3.2.2 需求点管理
- **优先级标注**：高/中/低三级
- **手动编辑**：支持修改需求点标题
- **删除功能**：支持删除单个需求点
- **数据联动**：删除需求点时自动删除关联的测试用例

### 3.3 测试用例生成模块

#### 3.3.1 生成模式
- **单点生成**：点击单个需求点，生成对应用例
- **批量生成**：一键为所有需求点生成用例
- **流式输出**：AI 生成采用 SSE 流式输出，前端打字机式渲染

#### 3.3.2 BDD 格式
每个测试用例包含：
- **Given（前置条件）**：执行用例的前置条件
- **When（操作步骤）**：具体的操作步骤
- **Then（预期结果）**：期望的输出结果

#### 3.3.3 用例属性
- **用例 ID**：格式 `模块缩写-序号`
- **用例标题**：格式 `[优先级] 场景描述`
- **优先级**：P0/P1/P2 三级
  - P0：核心主流程，影响上线
  - P1：重要功能，高频分支场景
  - P2：边缘功能，异常边界

### 3.4 用例管理模块

#### 3.4.1 视图模式
- **列表视图**：树形层级展示，支持展开/收起
- **思维导图视图**：XMind 风格可视化展示
  - 横向布局：根节点在左侧，需求点向右展开
  - 节点自适应宽度：150-350px
  - 贝塞尔曲线连接线
  - 支持缩放、拖拽平移

#### 3.4.2 用例操作
- **查看详情**：展开查看 BDD 场景
- **编辑标题**：支持修改用例标题
- **删除用例**：支持删除单个用例
- **验证状态**：标记用例是否已验证

#### 3.4.3 详情面板
- **条件显示**：默认隐藏，点击后滑入
- **内容展示**：
  - 需求点：标题、描述、验收标准、优先级
  - 测试用例：用例ID、标题、优先级、BDD 场景

#### 3.5.2 节点层级与属性

| 层级 | 节点类型 | 显示内容 | 优先级标签 | 节点宽度 |
|------|----------|----------|-----------|----------|
| Level 0 | 根节点 | 需求标题（截断20字符） | 无 | 标题长度×8 + 40（最小150，最大350） |
| Level 1 | 需求点 | 需求标题 | 高/中/低 | 标题长度×8 + 140 |
| Level 2 | 测试用例 | [优先级] 标题 | P0/P1/P2（圆形徽章） | 标题长度×8 + 100 |

#### 3.5.3 节点高度配置
```
根节点：56px（最高）
需求点：52px
测试用例：44px（最矮）
```

#### 3.5.4 间距配置
```
水平间距（GAP_X）：60px
垂直间距（GAP_Y）：24px
边距（PADDING）：40px
```

#### 3.5.6 交互规则

| 操作对象 | 点击效果 | 说明 |
|----------|----------|------|
| 根节点 | 切换视图 | 从思维导图视图切换到列表视图 |
| 需求点 | 展开/收起 | 展开显示所有子测试用例，收起只显示需求点 |
| 测试用例 | 显示详情 | 右侧滑出详情面板，展示 BDD 场景 |

#### 3.5.7 连接线样式
- **类型**：水平贝塞尔曲线
- **特点**：
  - 根节点到需求点：单条曲线
  - 需求点到测试用例：多条水平曲线
  - 视觉：从左到右横向展开

#### 3.5.8 视图缩放与平移
- **缩放**：鼠标滚轮控制缩放比例
- **平移**：拖拽空白区域移动视图
- **初始状态**：自动适应屏幕大小

#### 3.5.9 列表视图交互
- **展开/收起按钮**：每个有子节点的节点左侧显示
- **序号显示**：
  - 需求点显示需求点序号
  - 测试用例显示用例序号
- **优先级**：所有非根节点显示优先级标签
- **操作按钮**：悬停时显示（编辑、删除、生成用例）

#### 3.5.10 展开/收起控制
- **一键展开**：列表视图顶部按钮，展开所有需求点
- **一键收起**：列表视图顶部按钮，只保留根节点展开
- **记忆功能**：展开状态保存到 localStorage，刷新后恢复

#### 3.5.11 图例说明
思维导图左下角显示图例：
```
图例
● 根节点（紫色）
● 需求点（蓝色）
● 测试用例（灰色）
```

#### 3.5.12 数据联动
- **删除需求点**：同时删除该需求点下的所有测试用例
- **编辑需求点**：只修改标题，不影响子节点
- **生成用例**：为指定需求点生成测试用例，追加到该需求点下

### 3.6 数据导出模块

#### 3.5.1 CSV 导出
- 纯文本格式
- 字段：需求点、用例ID、标题、优先级、Given、When、Then

#### 3.5.2 Excel 导出
- 带格式和样式
- 分 Sheet：需求点列表、测试用例列表
- 表头冻结、筛选功能

#### 3.5.3 XMind 导出
- 标准 XMind 8 XML 格式（ZIP 压缩包）
- **文件结构**：
  ```
  .xmind 文件（ZIP格式）
  ├── mimetype              # 文件类型标识
  ├── content.xml           # 思维导图内容
  ├── metadata.xml          # 元数据
  └── styles.xml            # 样式定义
  ```
- **节点结构**：
  ```xml
  <topics type="main">
    <topic id="topic-root">        <!-- 根节点：需求标题 -->
      <title>需求标题</title>
      <children>
        <topics type="attached">  <!-- 需求点列表 -->
          <topic id="topic-REQ-001">
            <title>需求点1标题</title>
            <children>
              <topics type="attached">  <!-- 测试用例列表 -->
                <topic id="topic-TC-001">
                  <title>[P0] 测试用例标题</title>
                  <notes>
                    <plain-text-content>
                      【前置条件 Given】
                      ...

                      【操作步骤 When】
                      ...

                      【预期结果 Then】
                      ...
                    </plain-text-content>
                  </notes>
                </topic>
              </topics>
            </children>
          </topic>
          ...
        </topics>
      </children>
    </topic>
  </topics>
  ```
- **测试用例详情**：存储在 `<notes><plain-text-content>` 标签中，使用 BDD 格式

### 4.4 数据结构

#### 4.4.1 MindMapNode（思维导图节点）

```typescript
interface MindMapNode {
  id: string;                    // 唯一标识
  type: 'root' | 'requirement' | 'testcase';  // 节点类型
  title: string;                 // 标题
  description?: string;          // 描述（需求点）
  priority: '高' | '中' | '低' | 'P0' | 'P1' | 'P2';  // 优先级
  children: MindMapNode[];       // 子节点
  collapsed?: boolean;           // 是否折叠
  // BDD 场景（测试用例）
  given?: string;               // 前置条件
  when?: string;                // 操作步骤
  then?: string;                // 预期结果
}
```

#### 4.4.2 ParsedContent（已解析内容）

```typescript
interface ParsedContent {
  id: string;          // 唯一标识
  source: string;      // 来源：'attachment' | 'link' | 'manual'
  name: string;        // 文件名或链接标题
  content: string;     // 解析后的文本内容
}
```

#### 4.4.3 KnowledgeNode（知识库节点）

```typescript
interface KnowledgeNode {
  id: string;           // 唯一标识
  type: 'folder' | 'case';  // 节点类型
  title: string;       // 标题
  content?: string;    // 用例内容
  children?: KnowledgeNode[];  // 子节点（文件夹）
  caseCount?: number;  // 用例数量统计
}
```

## 6. 用户交互流程

### 6.1 完整用例生成流程

1. **输入需求**
   - 输入需求标题和描述
   - 或上传附件/抓取链接
   - 或从知识库选择已有用例

2. **拆分需求**
   - 点击"智能拆分"按钮
   - AI 自动分析并生成需求点树
   - 查看拆分结果，必要时手动调整

3. **生成用例**
   - 点击需求点右侧"生成用例"
   - 或一键"为所有需求点生成"
   - 查看 AI 流式输出的用例

4. **管理用例**
   - 切换列表/思维导图视图
   - 展开查看用例详情
   - 编辑/删除/标记验证状态

5. **导出数据**
   - 选择导出格式（CSV/Excel/XMind）
   - 下载文件用于其他工具

### 6.2 思维导图交互

| 操作 | 效果 |
|------|------|
| 点击需求点 | 展开/收起子用例列表 |
| 点击测试用例 | 显示详情面板 |
| 鼠标滚轮 | 缩放视图 |
| 拖拽空白区域 | 平移视图 |
| 点击节点标题 | 编辑节点 |

## 9. 注意事项

### 9.1 已知限制
- PDF 解析依赖服务端 Python 环境
- 飞书、Notion 等平台需手动导出后上传
- XMind 文件需使用 XMind 8 及以上版本打开

### 9.2 数据存储
- 本地数据存储在浏览器 localStorage
- 附件存储在 S3 对象存储
- 清理浏览器缓存会丢失本地数据

### 9.3 安全建议
- 敏感需求建议使用内网部署
- 定期导出重要用例数据
- 避免在公网环境输入机密需求
